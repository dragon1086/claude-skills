# 🧭 Tool Advisor

[English](./README.md) | [한국어](./README.ko.md) | **日本語** | [中文](./README.zh-CN.md) | [Español](./README.es.md) | [Português](./README.pt-BR.md) | [Русский](./README.ru.md) | [Français](./README.fr.md) | [Deutsch](./README.de.md)

**どのツールを使うべきか迷わないでください。Claude Codeにお任せください。**

Tool Advisorはプロンプトを分析し、最適なツール、エージェント、MCPサーバー、オーケストレーションパターンの組み合わせを推奨します。**コピー＆ペーストですぐに実行できるコマンド**も提供します。

![Demo](./assets/demo.gif)

## インストール

**方法1: ワンライナーインストール**
```bash
curl -fsSL https://raw.githubusercontent.com/dragon1086/claude-skills/main/install.sh | bash
```

**方法2: [skills.sh](https://skills.sh)経由**
```bash
npx skills add dragon1086/claude-skills
```

アップデートするには、同じコマンドを再度実行してください。

## 使い方

自然に質問するだけです：

```
"tool-advisorで分析して: 認証モジュールをリファクタリング"
"tool-advisor、このデータベース移行はどうすればいい？"
"ユーザー認証の実装に最適なアプローチは？"
```

**tool-advisor**と言及するか、ツールの推奨を求めると、スキルが有効になります。

## 機能

```
プロンプト入力
    ↓
┌─────────────────────────────────┐
│        Tool Advisor             │
├─────────────────────────────────┤
│ 1. ローカルインベントリ確認      │
│ 2. 複雑さの評価                 │
│ 3. ハーネスの必要性検出          │
│ 4. プランモード推奨？            │
│ 5. エージェント選択（3層）       │
│ 6. MCP必要性確認                │
│ 7. インストール提案              │
│ 8. クイックアクションテーブル    │
└─────────────────────────────────┘
    ↓
🎯 コピー＆ペーストで実行可能なコマンド
```

## 主な機能

| 機能 | 説明 |
|------|------|
| **クイックアクションテーブル** | すぐに実行できるコピペコマンド |
| **プランモード検出** | 複雑なタスクには`EnterPlanMode`を推奨 |
| **ハーネスパターン** | 「〜まで」「繰り返し」→ Ralphパターン |
| **3層エージェント検索** | ビルトイン → ローカル → マーケットプレイス |
| **MCP推奨** | インストール状況を表示（✅/❌）+ 代替案 |
| **ヒューマンインザループ** | 承認なしに絶対インストールしない |

## 例

**入力:**
```
tool-advisorで分析して: 認証モジュールをJWTにリファクタリング
```

**出力:**
```markdown
## 分析結果

### 1. 分類
- タイプ: リファクタリング/アーキテクチャ
- 複雑さ: 複雑（5+ファイル）

### 2. ハーネス
- 必要: いいえ

### 3. プランモード
- 推奨: はい
- 理由: 複数ファイル、セキュリティ関連

### 4. エージェント
- feature-dev（インストール済み）
- backend-developer（ローカルエージェント）
- security-engineer（ローカルエージェント）

### 5. 推奨
/feature-devをプランモードと併用

---

## 🎯 クイックアクション

| あなたの状況 | コピー＆ペースト |
|------------|----------------|
| まず計画を見たい | `JWTリファクタリングの計画を立てて` |
| ガイド付き開発 | `/feature-dev` |
| すぐ実装して | `認証をJWTにリファクタリングして` |

→ 推奨: 「まず計画」（5+ファイル、セキュリティ関連）
```
