# 🧭 Tool Advisor

[English](./README.md) | [한국어](./README.ko.md) | [日本語](./README.ja.md) | [中文](./README.zh-CN.md) | [Español](./README.es.md) | [Português](./README.pt-BR.md) | **Русский** | [Français](./README.fr.md) | [Deutsch](./README.de.md)

**Откройте свою среду инструментов. Усильте свои возможности.**

Tool Advisor сканирует вашу среду выполнения — MCP-серверы, навыки, плагины, CLI-инструменты — и предлагает оптимальные комбинации инструментов как **необязательные варианты** с готовыми командами для копирования. Он вооружает модель знаниями, которые она не получила бы иначе.

![Demo](./assets/demo.gif)

## Установка

**Способ 1: Однострочная установка**
```bash
curl -fsSL https://raw.githubusercontent.com/dragon1086/claude-skills/main/install.sh | bash
```

**Способ 2: Через [skills.sh](https://skills.sh)**
```bash
npx skills add dragon1086/claude-skills -y --agent claude-code
npx skills add dragon1086/claude-skills -y --agent codex
```

Для обновления выполните ту же команду снова.

## Использование

Просто спрашивайте естественно:

```
"Проанализируй с tool-advisor: рефакторинг модуля аутентификации"
"tool-advisor, как мне подойти к этой миграции базы данных?"
"Какой лучший подход для реализации аутентификации пользователей?"
```

Навык активируется, когда вы упоминаете **tool-advisor** или запрашиваете рекомендации по инструментам.

## Как это работает

```
Ваш промпт
    ↓
┌──────────────────────────────────┐
│     Tool Advisor v3.4            │
│  «Усилитель, не Командир»        │
├──────────────────────────────────┤
│ 1. Обнаружение среды              │
│    MCP / Навыки / Плагины / CLI  │
│ 2. Анализ задачи + Done When     │
│ 3. Сопоставление возможностей    │
│ 4. Предложение вариантов (до 3)  │
│ 5. Анализ пробелов               │
│ 6. Советы по производительности  │
└──────────────────────────────────┘
    ↓
Варианты + Таблица быстрых действий
```

## Возможности

| Функция | Описание |
|---------|----------|
| **4-уровневое сканирование среды** | Обнаруживает MCP-серверы, навыки, плагины и CLI-инструменты во время выполнения |
| **Критерии завершения** | Извлекает «Done when» из промптов, чтобы модель знала цель |
| **Адаптивный вывод** | Малые задачи <10 строк; крупные задачи — полный анализ |
| **Мульти-вариантные предложения** | До 3 подходов (Методичный / Быстрый / Глубокий) — никогда не навязывает |
| **Анализ пробелов** | Предлагает недостающие инструменты с пометкой «выполнимо и без них» |
| **Iron Rules и анти-паттерны** | 9 граничных правил предотвращают дрейф от советника к исполнителю; 3 анти-паттерна |
| **Принудительное выполнение фаз** | Каждая фаза ДОЛЖНА выдать результат или явное «N/A» — пропуск невозможен |
| **Human-in-the-loop** | Никогда не устанавливает без вашего одобрения |

## Пример

**Ввод:**
```
Проанализируй с tool-advisor: Рефакторинг модуля auth для использования JWT токенов
```

**Вывод:**
```markdown
## Tool Advisor v3.4 — Анализ среды и композиции

Prompt: `Рефакторинг модуля auth для использования JWT токенов`

### Ваша среда

| Уровень | Доступно |
|---------|---------|
| MCP-серверы | lsp, context7 |
| Навыки | /feature-dev, /code-review, /tdd |
| Плагины | oh-my-claudecode (33 агента) |
| CLI | git, node, pytest, docker |

### Профиль задачи
- **Тип**: Модификация / **Масштаб**: Большой / **Характеристики**: Требует планирования, есть тесты
- **Done when**: сессионная аутентификация заменена на JWT, тесты проходят, нет импортов сессий

### Релевантные возможности
- `lsp_diagnostics` — Проверка типов после изменений
- `ast_grep_search` — Поиск паттернов использования сессий
- `/feature-dev` — Управляемый рабочий процесс разработки

### Предложенные подходы

**A — Методичный** (Рекомендуется)
Task(Explore) -> EnterPlanMode -> Edit поэтапно -> Bash(pytest)

**B — Через навык**
/feature-dev (управляет всем процессом)

**C — Параллельные агенты**
[Task(Explore, bg), WebSearch("JWT best practices")] -> планирование -> реализация

---

## Быстрые действия

| Подход | Скопировать и вставить |
|--------|------------------------|
| Методичный | Начните с `Task(Explore)` для исследования кодовой базы |
| Через навык | `/feature-dev` и опишите рефакторинг |
| Быстрый | `Glob("**/auth/**") + Grep("session")` параллельно |

-> Рекомендуется: «Методичный» (крупный рефакторинг выигрывает от планирования)
```
